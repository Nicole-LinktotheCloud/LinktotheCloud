## Overview

A client needed to connect their Power Platform environment to external services with no prebuilt Microsoft connectors available. Bringing in prebuilt third-party connectors wasn't an option — the APIs required custom authentication and specific action mapping.

**Work included:**

- Custom connector development from scratch: OpenAPI (Swagger) definition, OAuth/API key authentication configuration, action and trigger mapping
- Testing each action against the live API with representative payloads
- Integration into the client's existing Power Apps and Power Automate flows
- Thorough documentation — connector reference, auth setup, and usage examples — written so the client could extend and maintain independently

The repeat engagement on this client noted documentation quality as a specific reason for return work.

**Outcome:** 5.0 rating · repeat client · noted for documentation quality
